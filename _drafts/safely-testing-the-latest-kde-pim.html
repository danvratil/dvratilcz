Do you want to see if your problem in KDE PIM has been already fixed? Do you want to help the KDE PIM developers to spot bugs and regressions early in the development cycle before they get into a stable release? You can do all of this and more without affecting your stable KDE PIM installation, keeping your emails safe and sound :)

Thanks to Aleix's work on Flatpak in KDE, we have a manifest (or a recipe, if you will) to build a Kontact Flatpak from latest source code with just a few commands.

You can find detailed instructions on how to get and build any application on the Flatpak KDE wiki.

First, make sure you have the KDE runtime installed:

flatpak remote-add kderuntime --from http://distribute.kde.org/kderuntime.flatpakrepo
flatpak install kderuntime org.kde.Platform//5.9
flatpak install kderuntime org.kde.Sdk//5.9

Then clone the repository with recipes for various KDE applications, and build Kontact:

git clone anongit.kde.org/flatpak-kde-applications.git
cd flatpak-kde-applications
mkdir app repo
flatpak-builder --ccache --repo=repo --subject="Build of Kontact `date`" app org.kde.kontact.json

Now go make a coffee. This is going to take a while.

Finally, add the local "repo" with the Kontact build to Flatpak and install Kontact from it:

flatpak remote-add kontact-repo repo --no-gpg-verify
flatpak install kontact-repo org.kde.kontact

You can re-run the flatpak-builder command followed by flatpak update whenever you want to update the Flatpak to the latest development version.
